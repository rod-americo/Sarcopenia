# Contributing to Heimdallr

Thank you for your interest in contributing to Heimdallr! We welcome contributions from the community.

## How to Contribute

1. **Open an Issue**: Before submitting a pull request, please open an issue to discuss proposed changes. This helps ensure your contribution aligns with the project's goals and avoids duplicate work.

2. **Fork and Branch**: Fork the repository and create a feature branch for your changes.

3. **Follow Code Style**: Maintain consistency with the existing codebase. Use clear variable names, add comments where necessary, and follow Python PEP 8 style guidelines.

4. **Test Your Changes**: Ensure your changes don't break existing functionality. Test the pipeline end-to-end if possible.

5. **Submit a Pull Request**: Once your changes are ready, submit a pull request with a clear description of what you've changed and why.

## Code of Conduct

Please be respectful and constructive in all interactions. We're building tools to improve patient care, and we value collaboration and professionalism.

## Questions?

If you have questions about contributing, please open an issue or reach out to the maintainers.
